## 1. Что это за проект

Это **веб-платформа для дистанционного изучения информатики** русскоязычными и, в первую очередь, *иностранными* студентами подготовительного факультета. Материал построен на реальном печатном пособии:

> «Рабочая тетрадь для иностранных студентов подготовительного факультета»
> Е. В. Степаненко, И. Т. Степаненко, Е. А. Нивина. — Тамбов: Изд-во ФГБОУ ВО «ТГТУ», 2018.

По содержимому `README.md` («Magistrer Diss») проект оформлен как **магистерская диссертационная работа** — то есть это учебный/квалификационный продукт, а не коммерческая система.

## 2. Технологический стек

Сознательно **минималистичный, «плоский» стек без фреймворков и без базы данных**:

- **Backend:** «голый» PHP, только `include`/`require`. Нет Composer, нет `package.json`, нет ORM, нет БД. Контент хранится прямо в `.php`-файлах как размеченный HTML.
- **Frontend:** Bootstrap 5 (лежит локально, не с CDN) + два собственных CSS-файла + два собственных JS-файла на чистом ванильном JavaScript (без jQuery/React).
- **Хранение прогресса:** `localStorage` в браузере (никакого сервера/аккаунтов).
- **Медиа:** 155 изображений (JPG/PNG), PDF-учебник, DOCX, видео (в репозиторий не включено — см. `.gitignore`).
- **Хостинг:** судя по `.gitignore` — обычный shared-хостинг с cPanel и загрузкой через Sprut.IO.

**Плюс такого выбора:** проект разворачивается за минуту на любом PHP-хостинге простым копированием файлов — не нужно ни сборки, ни установки зависимостей, ни миграций.

## 4. Как это работает (поток запроса)

Архитектура — простой **«фронт-контроллер на include»**:

1. `index.php` подключает `header.php`, выводит титул курса и три кнопки: «Начать курс», «Дополнительные занятия», «Открыть учебник (PDF)», затем `footer.php`.

2. **Основной курс:** `lessons.php?n=7` читает номер, **жёстко ограничивает его диапазоном 1–24** и подключает соответствующий `data/lesson7.php` внутрь общей обёртки (навбар, прогресс-бар, кнопки «назад/вперёд», подсветка активного номера). Если файл не найден — показывает аккуратный alert, а не «ломается».

3. **Доп. занятия:** `extra-lessons.php` хранит массив-белый-список уроков (`autocad`, `office-test`); без `?id` показывает карточки-список, с `?id=slug` — конкретный урок.

4. **Условная подгрузка тяжёлого JS:** файлы `pascal-trainer.css` и `pascal-trainer.js` подключаются **только для занятий 19–24** (`if ($lesson >= 19 && $lesson <= 24)`), чтобы не грузить интерпретатор Pascal на «словарных» уроках. Это грамотная оптимизация.

5. **Прогресс** сохраняется в `localStorage`: `lastLesson` (последнее занятие) и `lesson_{N}_completed` (24 флага) → верхний прогресс-бар считает процент пройденного курса.

## 5. Педагогическая модель и типы контента

Курс чётко делится на **два блока** с разной дидактикой.

### Занятия 1–18 — информатика как «русский язык специальности» (RFL)

Это не столько программирование, сколько **изучение русскоязычной терминологии информатики** для иностранцев:

- Словарные таблицы с **ударениями** (Компью́тер, Клавиату́ра, Трансля́тор).
- Кнопки **озвучки** (▶) у каждого термина — синтез речи прямо в браузере. Всего по сайту таких кнопок **777**.
- Картинки из рабочей тетради (сконцентрированы в занятиях 1–6; дальше темы всё более абстрактные — алгоритмы, кодирование — и картинок почти нет).
- Проверочные задания: «да/нет», тесты с выбором ответа, группировка однокоренных слов, drag & drop.

Тематически (по `pdf-page-map.txt`) это путь: устройство компьютера → информация и кодирование → файлы → алгоритмы и блок-схемы → Windows / Paint / Word / Excel.

### Занятия 19–24 — язык Паскаль с интерактивными тренажёрами

Здесь появляется настоящее программирование и **исполняемый код в браузере** (см. раздел 7).

### Дополнительные занятия

- **AutoCAD** — пошаговый дистанционный урок (интерфейс, примитивы, привязки, экспорт), со скриншотами и видео.
- **Тест «Word + Excel»** — онлайн-тест с выбором ответа и подсчётом результата.

## 6. Интерактивное ядро — `assets/js/main.js` (~826 строк)

Хорошо структурированный, подробно закомментированный (по-русски) движок на **чистом JavaScript**. Задания «подключаются» через CSS-классы и `data-`атрибуты в HTML, а `main.js` сам всё инициализирует по `DOMContentLoaded`. Реализовано:

1. **Озвучка** (`.speak-btn`) через `SpeechSynthesis`, с поиском русского голоса и делегированием событий (работает и для динамически добавленных кнопок).
2. **Вопросы «да/нет»** с пояснениями.
3. **Drag & drop** — с полноценной **поддержкой тач-устройств** (перетаскивание на телефоне, блокировка прокрутки только во время перетаскивания).
4. **Проверка `<select>` и `<input>`** с «мягкой» нормализацией ответа (ё→е, лишние пробелы, регистр).
5. **«Соедини пары»** (tap-tap) с перемешиванием вариантов.
6. **MCQ** (тесты с выбором) с подсчётом баллов и подсветкой.
7. **Группировка слов по корням** (клик-механика: выделить слово → положить в корзину).
8. **Глобальный прогресс** по 24 занятиям.

**Отдельно отмечу инженерную аккуратность:** почти везде `localStorage` обёрнут в `try/catch` (не падает в приватном режиме), используется делегирование событий, есть хелпер плавной прокрутки к результату проверки.

## 7. Технический центр проекта — `assets/js/pascal-trainer.js` (~815 строк)

Это **самая сильная и нетипичная часть работы**: собственный **интерпретатор Паскаля, написанный вручную на JavaScript**. Такое редко встретишь в студенческих проектах — обычно берут готовую библиотеку. Здесь всё своё:

### 7.1. Подсветка синтаксиса
Регулярками, с приёмом «прятанья токенов» (`\u0001S{n}\u0001`), чтобы уже подсвеченные фрагменты не подсвечивались повторно. Отдельные классы для ключевых слов, типов, встроенных процедур, строк, чисел, комментариев, операторов.

### 7.2. Полноценный конвейер «лексер → парсер → интерпретатор»
- **Лексер** превращает код в токены (числа, строки с экранированием `''`, идентификаторы, ключевые слова, операторы, комментарии `{}`, `(* *)`, `//`).
- **Парсер рекурсивного спуска** строит AST с корректной иерархией приоритетов операций (`expr → simpleExpr → term → factor`).
- **Интерпретатор** обходит дерево и выполняет программу.

### 7.3. Что поддерживается
`program/var/const/begin/end`, присваивание `:=`, арифметика `+ - * /`, `div`, `mod`, `writeln/write`, циклы `for … to/downto … do`, `while`, `repeat … until`, ветвление `if/then/else`, **одномерные массивы**, `boolean`, встроенные функции (`abs, sqr, sqrt, sin, cos, round, trunc, length, chr, ord, random`).

### 7.4. Что НЕ поддерживается (честно задокументировано автором)
Процедуры и функции пользователя, записи (`record`), операции над строками, файловый ввод-вывод, `readln` (ввод игнорируется). Для вводного курса этого достаточно.

### 7.5. Защита от «зависания»
Есть счётчик итераций с лимитом **50 000** — бесконечный цикл не подвесит вкладку, а выдаст понятную ошибку. Это важная деталь безопасности для «кода в браузере».

### 7.6. Четыре типа тренажёров
1. **▶ Запустить** — код выполняется, вывод показывается под ним (+ кнопка «Копировать»).
2. **Что выведет?** — студент вписывает предполагаемый вывод, проверка сравнивает с эталоном (с нормализацией пробелов/регистра).
3. **Заполни пропуск** — `<input>` прямо внутри листинга; проверка по `data-correct`.
4. **Расставь по порядку** — drag & drop строк. Умная логика: строки с **одинаковым** `data-order` считаются взаимозаменяемыми (например, два независимых присваивания можно ставить в любом порядке) — правильным считается любой **неубывающий** порядок. Это методически грамотно: учит понимать зависимости между операторами, а не зубрить одну «правильную» последовательность.

### 7.7. Пограничные семантические нюансы (не баги, но знать полезно)
- `mod` в интерпретаторе реализован как **всегда неотрицательный** остаток (`((a%b)+b)%b`), тогда как в вычислении констант — обычный `%`. Классический Паскаль для отрицательных чисел ведёт себя иначе. На положительных числах (а это 99% учебных примеров) расхождения нет.
- `and`/`or` реализованы как **логические** (для `boolean`), но не как побитовые для целых. Для учебных примеров это корректно, для `5 and 3` — уже нет.

## 9. Безопасность

Для «плоского» PHP-сайта посмотрел на главный риск — **включение файлов (LFI)** через параметры URL. Здесь всё сделано правильно:

- `lessons.php` приводит `?n` к `(int)` и **зажимает в диапазон 1–24** до подстановки в путь — произвольный файл подключить нельзя.
- `extra-lessons.php` работает по **белому списку** ключей (`$extraLessons[$id]`) — тоже безопасно.
- Пользовательский вывод в доп-уроках прогоняется через `htmlspecialchars()`.

**Один нюанс дидактического характера:** в тесте «Word + Excel» правильные ответы попадают в HTML как `data-correct="…"`, а подсчёт идёт на клиенте (JavaScript). Для **самопроверки** это нормально и распространено, но для реального оценивания ответы «видны» через исходный код страницы.

## 10. Сильные стороны

1. **Нулевые зависимости и портируемость.** Копируешь папку на любой PHP-хостинг — работает. Ни сборки, ни БД, ни настройки.
2. **Собственный интерпретатор Паскаля** — серьёзная, самостоятельная инженерная работа, выделяющая проект на фоне типичных «HTML+CSS» магистерских.
3. **Продуманная дидактика:** озвучка с ударениями для иностранцев, разнообразные форматы заданий, «умная» проверка порядка строк, мгновенная обратная связь.
4. **Внимание к мобильным и доступности:** тач-drag&drop, `aria-label`, адаптивные брейкпоинты, «мягкая» проверка ввода.
5. **Аккуратный код фронтенда:** делегирование событий, `try/catch` вокруг `localStorage`, условная загрузка тяжёлого JS только там, где он нужен.
6. **Безопасная работа с include** (int-clamp + белый список).
7. **Хорошая внутренняя документация** (комментарии, `structure.txt`, карта страниц PDF).

## 11. Слабые стороны и что улучшить

1. **Разнобой в названиях** (ТГТУ / ПГУ / penzarepetitor / «Magister Diss») — привести к единому бренду; переписать пустой `README.md` (сейчас там две строки).

Каждый урок — огромный рукописный HTML (`lesson6.php` — 2968 строк!), всего ~21 000 строк в `data/`. Просится вынесение словарей/заданий в данные (JSON/массивы) и генерация разметки шаблоном.
3. **«Кухня» разработчика в репозитории.** Папка `.dev-notes/backup/` с десятками `.bak`/`.bak5` — это версионирование «руками» поверх Git. Стоит удалить из репозитория (для этого и нужен сам Git).
4. **Повторы в CSS.** `.word-grouping .word-bank` и `.l3-placeholder` объявлены дважды с конфликтующими значениями (`color: #212529 !important` vs `#6c757d`). Нужно консолидировать.
5. **Ответы теста видны на клиенте** (см. раздел 9) — если тест станет «зачётным», проверку надо перенести на сервер.
6. **Неиспользуемые возможности.** `match-pairs` задействован лишь в 1 уроке, `input[data-answer]` — почти нигде. `main.js` тащит код, который почти не работает: либо применить шире, либо убрать.
7. **Нет автотестов и лицензии.** Для интерпретатора Паскаля напрашивается небольшой набор тестов (эталонные программы → ожидаемый вывод). Отсутствие файла лицензии оставляет правовой статус кода неопределённым.
8. **Абсолютные пути к ресурсам** (`/assets/...`) требуют, чтобы сайт лежал в корне домена, а не в подпапке — небольшое ограничение развёртывания.

## Итог одним абзацем

`Magister` — это аккуратно сделанная, самодостаточная обучающая веб-платформа по информатике для иностранных студентов подготовительного факультета, оформленная как магистерская работа. Технологически она намеренно простая (PHP + Bootstrap + ванильный JS, без БД и сборки), что делает её очень портируемой. Дидактически — сильная: озвучка терминов с ударениями, разнообразные интерактивные задания и, главное, **собственный интерпретатор Паскаля прямо в браузере** с четырьмя типами тренажёров и защитой от зацикливания. Основные направления доработки — не в функциональности, а в «гигиене» проекта: убрать дублирование и рукописные бэкапы, вынести контент уроков в данные, унифицировать название и добавить тесты с лицензией.
