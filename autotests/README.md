# Autotests — Magister

Браузерные автотесты сайта на **Playwright + Firefox**. Бот открывает занятия и
проходит их **как студент — сверху вниз, читая и отвечая**: курсор движется вниз
по странице, без скачков вверх-вниз. В словарных уроках просто кликает слова
(озвучка), в упражнениях вводит верные ответы.

Один движок, два режима запуска:

- **graphical** — видно окно браузера и курсор бота (красная точка);
- **console** — без окна (headless), отчёт в консоль и в JSON.

## Философия проверки

Тест — не придирка к разметке, а проверка, что **сайт работает, когда студент
отвечает правильно**. Поэтому ПРОВАЛ засчитывается только на реальных признаках
поломки:

- сайт пометил **верный** ответ как неверный (`is-invalid`, «Неправильно», красный feedback);
- JS-исключение или `console.error` на странице;
- запрос с кодом ≥ 400 (битый ресурс) или картинка, которая не отрисовалась;
- текст ошибки PHP на странице;
- интерпретатор Pascal вернул ошибку на «▶ Запустить».

Отсутствие ожидаемой подсветки провалом не считается — так тест не «фолзит» на
страницах с нестандартными виджетами, которые на самом деле работают.

## Требования

- **Node.js 18+** в PATH.
- **PHP** — portable в `..\MagisterTest\vendor\php\php.exe`, системный `php` в PATH
  (XAMPP/Laragon/WAMP/OSPanel) или путь в `MAGISTER_PHP`.

Firefox для Playwright и npm-зависимости BAT-файлы ставят сами при первом запуске.

## Запуск

Двойным щелчком или из консоли:

```bat
graphical.bat      REM видно окно и курсор бота
console.bat        REM без окна, отчёт в консоли
```

Оба BAT принимают те же флаги, что и `run.mjs`:

```bat
console.bat --lessons 1,19-24 --no-office
graphical.bat --lessons 6
```

## Флаги `run.mjs`

| Флаг | Значение | По умолчанию |
|---|---|---|
| `--lessons` | `all`, диапазон `1-24`, список `1,5,7` | `all` |
| `--office` / `--no-office` | включать ли тест Word/Excel | включён |
| `--browser` | `firefox`, `chromium`, `webkit` | `firefox` |
| `--headed` | показать окно и курсор | выкл. (headless) |
| `--mouse-delay` | пауза (мс) после подведения курсора | 120 headed / 0 headless |
| `--slowmo` | замедление Playwright (мс) | 30 headed / 0 headless |
| `--speak-limit N` | ограничить число кликов по озвучке за урок | без лимита |
| `--check-negative` | доп. тихая проверка реакции на **неверный** ответ (без движения мыши) | выкл. |

Проверка «неверный ответ подсвечивается как неверный» по умолчанию **выключена**,
чтобы не дёргать курсор туда-сюда. С флагом `--check-negative` она выполняется
программно (значение меняется без мыши) и не мешает плавному проходу.

Переменные окружения: `MAGISTER_SITE`, `MAGISTER_PHP`, `MAGISTER_AUTOTEST_PORT`.

## Что делает бот на занятии

Собирает все задания в порядке их расположения на странице и идёт по ним сверху
вниз одним проходом:

- **озвучка** — кликает `.speak-btn` (словарь: «читаем» слова);
- **да/нет** — кликает подпись верного варианта (radio скрыт под `btn-check`);
- **ввод текста** (`input[data-answer]`) — вводит верный ответ;
- **выпадающие списки** мгновенной проверки — выбирает верный вариант;
- **MCQ-наборы** — отмечает верные и жмёт «Проверить»;
- **задачи с кнопкой «Проверить»** (task2/3/4 и т.п.) — заполняет корректные
  значения и жмёт кнопку (best-effort, без строгой оценки);
- **Pascal** (19–24) — «▶ Запустить», «Что выведет?» (эталон), «Заполни пропуск»;
- **тест Word/Excel** — все верные → ждёт 100 %.

Drag&drop и «соедините пары» не трогаются строго (сложная ручная механика) — они
покрыты статически в наборе MagisterTest.

## Отчёты

`reports/report-<режим>-<браузер>.json` — по каждой цели: сколько заданий отвечено,
сколько слов озвучено, список провалов, console-ошибки, неудачные запросы. Код
выхода `1`, если хоть одна цель провалена (для CI). `reports/` и `node_modules/`
— в `.gitignore`.

## Связь с MagisterTest

`..\MagisterTest\` — отдельный тяжёлый QA-стенд на Python/pytest: статическая
проверка структуры и ссылок, целостность банка вопросов (518 заданий), HTTP-тесты
и собственные браузерные тесты, история прогонов в SQLite. Эти autotests — лёгкая,
не требующая Python браузерная проверка «глазами студента» прямо в репозитории
сайта. Логика поиска PHP и селекторы согласованы между наборами.
